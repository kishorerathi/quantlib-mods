QuantLib CVS Repository

If you're new to CVS please read our FAQ and SourceForge.net CVS documentation, especially their CVS FAQ. For general (and more advanced) information consult the free CVS Book and the CVS Docs at www.loria.fr

QuantLib CVS repository includes the following modules:

  • QuantLib (the C++ library)
  • QuantLib-SWIG (bindings for Python, Ruby, MzScheme and Guile.)
  • QuantLibXL (the Excel add-in)
  • QuantLib-site (the HTML sources of these pages)

Please note that QuantLib modules have capital Q and capital L.

You can browse the CVS tree: this gives you a view into the current status of the project's code. You may also view the complete histories of any file in the repository.

Anonymous CVS Access

The modules can be checked out through anonymous (pserver) CVS with the following instruction set. The module you wish to check out must be specified as the modulename. When prompted for a password for anonymous, simply press the Enter key.

Unix:

export CVSROOT=:pserver:anonymous@cvs.quantlib.org:/cvsroot/quantlib
cvs login
cvs -z3 co -P modulename

Win32:

SET CVSROOT=:pserver:anonymous@cvs.quantlib.org:/cvsroot/quantlib
cvs login
cvs -z3 co -P modulename

After initial checkout, you can execute cvs commands in this directory without the CVSROOT exporting/setting. For example:

cvs update

If you are behind a firewall ask your sysadmin to open the cvspserver port (2401) for outgoing connection from your machine to cvs.quantlib.org
Alternatively, if you are behind a strict firewall, you can download the nightly backup tarball of the CVS repository. You'll need to install it as a local CVS repository to use it. However, we advise against such operation as the tarball size exceeds 60 MB at the time of this writing.

Finally, another possibility (untested at this time) might be to use an http grabber such as CVSGrab or sfcvs. We'll be grateful to anyone reporting success or failure with these packages.


Developer CVS Access via SSH

If you are a SourceForge.net-recognized QuantLib developer you can access the CVS tree via SSH (SSH must be installed on your client machine). To check if this is the case try the shell login. Substitute developername with the proper value:

ssh -l developername quantlib.org
groups

You have to perform the shell login at least once, in order to have your user directory created. If the shell login is OK (type exit to log off), you can check out modules from the QuantLib CVS. Substitute modulename and developername with the proper values, enter your site password when prompted:

Unix:

export CVSROOT=developername@cvs.quantlib.org:/cvsroot/quantlib
export CVS_RSH=ssh
cvs -z3 co -P modulename

Win32:

SET CVSROOT=developername@cvs.quantlib.org:/cvsroot/quantlib
SET CVS_RSH=ssh
cvs -z3 co -P modulename

After initial checkout, you can change into this directory and execute cvs commands without the CVSROOT exporting/setting. For example:

cvs update
cvs commit -m "comments for this commit"
cvs add myfile.c

If you are behind a firewall you need to open the ssh port (22) for outgoing connection from your machine to quantlib.org, shell.sf.net, and upload.sf.net (the latter just in case you plan to help the QuantLib release process).

Note for C++ Unix users: if you use CVS, you will need some GNU tools that usually only developers use, and which are not required to build QuantLib from tarballs. These are automake, autoconf, libtool, GNU m4, GNU make, and others which might escape me now. They all come with recent GNU/Linux distributions.

To begin the build process from a CVS checkout, start with:

sh ./autogen.sh

which will prepare the package for compilation. You can then use ./configure and make in the usual way.
